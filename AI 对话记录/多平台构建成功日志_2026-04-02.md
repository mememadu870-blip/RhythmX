## 问题诊断与修复过程

### 阶段 1: 添加缺失的导出预设

**问题**: GitHub 上 Linux 和 Web 构建失败，提示导出预设不存在

**原因**: `export_presets.cfg` 文件中只有 "Windows Desktop" 和 "Android" 两个预设，缺少 "Linux/X11" 和 "Web" 预设

**修复**: 手动添加 Linux/X11 和 Web 导出预设到 `export_presets.cfg`

```cfg
[preset.2]
name="Linux/X11"
platform="Linux/X11"
runnable=true
export_path="./RhythmX.x86_64"
binary_format/architecture="x86_64"

[preset.3]
name="Web"
platform="Web"
runnable=true
export_path="./RhythmX_web/index.html"
html/canvas_resize_policy=2
```

**提交**: `6423ad2 feat: add Linux/X11 and Web export presets and re-enable CI builds`

### 阶段 2: 导出模板路径问题

**问题**: Run 18-20 连续失败，Linux 和 Web 构建提示找不到导出模板

**错误信息**:
```
ERROR: Template not found for platform "linux_x86_64"
ERROR: Template not found for platform "web"
```

**原因分析**:
- `barichello/godot-ci:4.6` 容器中的模板存放位置与 Godot 4.x 的目录结构不匹配
- Godot 4.6 使用新的模板目录结构：`templates/4.6.stable/` 而非 `export_templates/4.6.stable/`
- 容器内模板位于 `/root/.local/share/godot/` 但 Godot 运行时查找 `~/.local/share/godot/`

**修复方案**:
1. 增加详细的日志输出，查看容器内实际的模板目录结构
2. 同时复制 `templates/` 和 `export_templates/` 两个目录的内容
3. 确保目标路径与 Godot 4.6 的期望结构匹配

**关键修复代码**:
```yaml
- name: Setup Export Templates
  run: |
    mkdir -p ~/.local/share/godot/export_templates
    mkdir -p ~/.config/godot

    # Copy config
    if [ -d "/root/.config/godot" ]; then
      cp -r /root/.config/godot/* ~/.config/godot/ || true
    fi

    # Check and copy templates from containers
    if [ -d "/root/.local/share/godot/templates/${GODOT_VERSION_FULL}.stable" ]; then
      cp -r /root/.local/share/godot/templates/${GODOT_VERSION_FULL}.stable \
        ~/.local/share/godot/templates/
    fi

    # Also copy export_templates (older structure)
    if [ -d "/root/.local/share/godot/export_templates/${GODOT_VERSION_FULL}.stable" ]; then
      cp -r /root/.local/share/godot/export_templates/${GODOT_VERSION_FULL}.stable \
        ~/.local/share/godot/export_templates/
    fi
```

## 经验总结

### Godot 4.x 导出模板结构变更

Godot 4.6 的导出模板存放位置与 4.2-4.3 不同：

| Godot 版本 | 模板路径 |
|-----------|---------|
| 4.2-4.3   | `export_templates/4.x.stable/` |
| 4.6+      | `templates/4.x.stable/` 优先，兼容 `export_templates/` |

### 容器化构建注意事项

1. **路径映射**: 容器内路径 `/root/` 需要复制到 `~/.local/share/godot/`
2. **模板完整性**: 同时检查 `templates/` 和 `export_templates/` 两个目录
3. **详细日志**: 在模板设置步骤添加 `ls -laR` 输出便于排查问题

### 导出预设命名

工作流中的预设名称必须与 `export_presets.cfg` 完全一致：
- ✅ `"Linux/X11"` (不是 `"Linux"`)
- ✅ `"Web"` (不是 `"HTML5"`)
- ✅ `"Windows Desktop"` (不是 `"Windows"`)
