## 🔧 主要技术问题与解决方案

### 问题 1: Godot 4.x 不支持 class_name + 内部类
**现象**: `SCRIPT ERROR: Could not preload resource script "res://scripts/data/GameData.gd"`

**原因**: Godot 4.x 无法正确处理包含 `class_name` 声明且有多个内部类的文件

**解决方案**:
```gdscript
# ❌ 原方案 (C# 风格嵌套类)
class GameData:
    class SongData:
        ...
    class ChartData:
        ...

# ✅ 新方案 (独立文件)
# scripts/data/types/SongData.gd
extends RefCounted
# SongData 实现...

# scripts/data/GameData.gd
const SongData = preload("res://scripts/data/types/SongData.gd")
const ChartData = preload("res://scripts/data/types/ChartData.gd")
```

**创建的类型文件** (10 个):
1. `scripts/data/types/SongData.gd` - 歌曲数据
2. `scripts/data/types/ChartData.gd` - 谱面数据
3. `scripts/data/types/NoteData.gd` - 音符数据
4. `scripts/data/types/SongRecord.gd` - 歌曲记录
5. `scripts/data/types/AchievementRecord.gd` - 成就记录
6. `scripts/data/types/AchievementDefinition.gd` - 成就定义
7. `scripts/data/types/PlayerData.gd` - 玩家存档数据
8. `scripts/data/types/LeaderboardEntry.gd` - 排行榜条目
9. `scripts/data/types/PlayRecordRequest.gd` - 游玩记录请求
10. `scripts/data/types/ScoreResult.gd` - 分数结果

### 问题 2: 循环依赖
**现象**: `GameData.gd` 需要 `GameManager.gd` 的枚举，`GameManager.gd` 需要 `GameData.gd` 的类型

**解决方案**: 创建独立的 `GameTypes.gd`
```gdscript
# scripts/game/GameTypes.gd
class_name GameTypes
extends RefCounted

enum Difficulty { EASY, NORMAL, HARD, EXPERT }
enum NoteType { TAP, HOLD, SWIPE }
enum SwipeDirection { NONE, LEFT, RIGHT, UP, DOWN }

static var DIFFICULTY_DENSITY_MULTIPLIERS: Array[float] = [0.4, 0.7, 1.0, 1.3]
static var DIFFICULTY_NAMES: Array[String] = ["Easy", "Normal", "Hard", "Expert"]
static var DIFFICULTY_COLORS: Array[Color] = [...]
```

### 问题 3: Godot 4.x API 变更
| C# / Godot 3.x | GDScript / Godot 4.x | 文件 |
|---------------|---------------------|------|

| `AudioStreamOGGVorbis` | `AudioStream` | AudioAnalysis.gd |

### 问题 4: 跨文件类型引用
**现象**: `Identifier "GameManagerClass" not declared in the current scope`

**原因**: GDScript 需要显式 `preload()` 才能访问其他文件的类型

**解决方案**: 在所有需要引用的文件中添加:
```gdscript
const GameManagerClass = preload("res://scripts/autoload/GameManager.gd")
const GameData = preload("res://scripts/data/GameData.gd")
const ScoreManagerClass = preload("res://scripts/autoload/ScoreManager.gd")
```

## 💡 经验总结

### Godot 4.x GDScript 最佳实践

1. **避免内类类嵌套**
   - 每个 `class_name` 类放在独立文件中
   - 使用 `preload()` 进行跨文件引用

2. **处理循环依赖**
   - 将共享枚举和常量提取到独立文件
   - 使用 `preload()` 而非 `class_name` 全局引用

3. **对象实例化**
   - GDScript 必须使用 `.new()` 实例化对象
   - `ClassName.new()` 而非 `ClassName()`

4. **类型系统**
   - Godot 4.x 移除了一些专用音频类型
   - 使用通用类型 `AudioStream` 替代 `AudioStreamOGGVorbis`

5. **错误处理**
   - GDScript 没有 C# 风格的 `try/catch`
   - 使用 `if` 检查和返回值处理错误
